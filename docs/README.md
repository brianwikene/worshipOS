# WorshipOS Documentation

This folder contains the philosophical, theological, architectural, and product foundations of WorshipOS.

These documents exist to preserve **intent**, not just implementation.
They are here so future contributors ‚Äî and future Brian ‚Äî do not have to reconstruct *why* decisions were made.

If code and documentation disagree, the code is wrong.

---

## What This Documentation Is

This is not a feature list.  
This is not marketing copy.  
This is not speculative design.

These documents define:
- the convictions that shape the product
- the boundaries that protect trust and care
- the architectural decisions that enforce those boundaries
- the features we will build ‚Äî and the ones we will not

WorshipOS is not neutral software.
It encodes pastoral, theological, and ethical commitments.

## Core Commitments (Summary)

WorshipOS is built on these commitments:

- People live in seasons, not stages
- There is a diversity of encounter with God
- Growth reflects formation, not qualification
- Tending is care, not control
- Metrics surface context, not success
- Tasks serve responsibility, not bureaucracy
- Truth is preserved, not polished
- History is appended, not erased
- Robust, easy, powerful, and contextual are all required ‚Äî not tradeoffs
